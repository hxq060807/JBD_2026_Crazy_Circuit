# 赛道参数输入交互改进计划

## 1. 任务概述

**目标**：改进赛道参数输入交互，允许用户自由切换和修改之前输入的节点参数，而不是按顺序输入，输错了就得全部重新输入。

**当前问题**：
- 目前的输入交互是按顺序进行的，用户必须按顺序输入每个节点
- 如果用户在输入过程中出错，必须全部重新输入
- 没有提供修改已输入节点的功能

**解决方案**：
- 实现一个节点管理界面，允许用户自由选择要查看和修改的节点
- 提供前后导航功能，方便用户在节点之间切换
- 保持原有的参数输入逻辑，同时增加修改功能

## 3. 实现思路

1. **节点管理界面**：
   - 显示已输入的节点列表
   - 提供导航按钮（上一个、下一个、修改、添加、退出）
   - 显示当前节点的详细信息

2. **导航功能**：
   - 使用键盘按键实现节点之间的切换
   - 支持循环导航（最后一个节点后回到第一个，第一个节点前回到最后一个）

3. **修改功能**：
   - 进入修改模式后，允许用户重新输入节点类型和参数
   - 修改完成后返回节点管理界面

4. **数据存储**：
   - 保持原有的Flash存储逻辑
   - 确保修改后的数据正确保存到结构体

## 4. 预期效果

用户可以：
1. 选择赛道并进入参数输入模式
2. 按顺序输入节点参数
3. 在任何时候暂停输入，查看已输入的节点
4. 自由切换到任意已输入的节点进行修改
5. 修改完成后继续输入新节点
6. 退出并保存所有参数

这样，用户就不再需要因为一个节点的输入错误而重新输入所有节点参数，大大提高了输入效率和用户体验。
