# 赛道里程记录与提前减速功能 - 实现计划

## [x] 任务 1：添加圈数计数和里程记录标志
- **优先级**：P0
- **依赖**：无
- **描述**：
  - 在Ctrl.h中添加圈数计数变量和里程记录标志
  - 在Ctrl.c中初始化这些变量
- **验收标准**：AC-1
- **测试要求**：
  - `programmatic` TR-1.1：检查圈数计数变量是否正确初始化
  - `programmatic` TR-1.2：检查里程记录标志是否正确设置
- **说明**：需要在全局变量中添加这些新的变量，用于跟踪当前圈数和是否正在记录里程

## [x] 任务 2：实现第一圈慢速行驶逻辑
- **优先级**：P0
- **依赖**：任务 1
- **描述**：
  - 修改Set_Speed函数，在第一圈时使用较低的基础速度
  - 确保赛车能够稳定行驶并准确检测节点
- **验收标准**：AC-1
- **测试要求**：
  - `human-judgment` TR-2.1：观察第一圈行驶速度是否明显低于后续圈数
  - `programmatic` TR-2.2：检查第一圈时的速度设置是否正确
- **说明**：第一圈的速度应该足够慢，确保赛车能够稳定通过所有节点，同时准确记录里程

## [x] 任务 3：实现里程记录功能
- **优先级**：P0
- **依赖**：任务 1, 任务 2
- **描述**：
  - 修改Mode_Choose和相关函数，在第一圈时记录每两个拐点节点之间的里程
  - 将记录的里程存储到Run_Track结构中
- **验收标准**：AC-1
- **测试要求**：
  - `programmatic` TR-3.1：检查里程数据是否正确存储到Run_Track结构中
  - `programmatic` TR-3.2：验证里程记录的准确性
- **说明**：需要在节点切换时计算并存储里程数据，确保数据的准确性和完整性

## [x] 任务 4：实现提前减速逻辑
- **优先级**：P0
- **依赖**：任务 3
- **描述**：
  - 修改Set_Speed函数，在第二圈及以后，当接近转弯节点时提前减速
  - 设置合适的减速阈值和减速程度
- **验收标准**：AC-2
- **测试要求**：
  - `human-judgment` TR-4.1：观察赛车在接近转弯节点时是否提前减速
  - `human-judgment` TR-4.2：观察赛车在转弯时是否稳定，无打滑和甩尾
  - `programmatic` TR-4.3：检查减速逻辑是否正确触发
- **说明**：需要根据记录的里程信息，在接近转弯节点时逐渐降低速度，确保赛车能够稳定通过转弯

## [x] 任务 5：实现圈数切换逻辑
- **优先级**：P1
- **依赖**：任务 1, 任务 3
- **描述**：
  - 实现圈数计数逻辑，当赛车完成一圈后增加圈数计数
  - 切换到正常速度行驶模式
- **验收标准**：AC-1, AC-2
- **测试要求**：
  - `programmatic` TR-5.1：检查圈数计数是否正确增加
  - `human-judgment` TR-5.2：观察赛车在第二圈时是否切换到正常速度
- **说明**：圈数切换点可以设置在赛道的起点或终点，需要确保准确检测完成一圈

## [x] 任务 6：优化减速参数
- **优先级**：P1
- **依赖**：任务 4
- **描述**：
  - 调整减速阈值和减速程度，以获得最佳的转弯性能
  - 确保减速过程平滑，避免突然减速导致的不稳定
- **验收标准**：AC-2
- **测试要求**：
  - `human-judgment` TR-6.1：观察赛车在转弯时的稳定性
  - `human-judgment` TR-6.2：评估减速过程是否平滑
- **说明**：减速参数需要根据实际赛道和赛车性能进行调整，可能需要多次测试来找到最佳值

## [x] 任务 7：测试和验证
- **优先级**：P1
- **依赖**：所有任务
- **描述**：
  - 在实际赛道上测试功能
  - 验证里程记录的准确性和提前减速的效果
  - 确保系统在多次行驶中表现一致
- **验收标准**：AC-1, AC-2, AC-3
- **测试要求**：
  - `human-judgment` TR-7.1：评估整体性能和稳定性
  - `programmatic` TR-7.2：检查系统在多次行驶中是否使用了存储的里程数据
- **说明**：需要在实际赛道上进行测试，确保功能正常工作并达到预期效果
